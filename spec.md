# Specification: PionBridge (Flutter-Go WebRTC Sidecar)

## 1. Executive Summary

**PionBridge** is a cross-platform Flutter library that provides P2P capabilities by wrapping the **Pion WebRTC** (Go) stack. It operates by running a local Go-based sidecar server on the device, which communicates with the Flutter Dart layer via a **single persistent WebSocket connection** using **MessagePack serialization**. This architecture bypasses the limitations of traditional platform channels for high-frequency or complex WebRTC state management.

**Key Principle:** The Go server is stateful and manages all WebRTC objects. Dart is a thin proxy that sends operations and listens to events over a single, efficient binary WebSocket connection.

---

## 2. System Architecture

### 2.1 Component Stack

* **Go Core:** Handles the WebRTC stack (Pion library), ICE gathering, and SDP signaling. Manages a registry of active resources.
* **Platform Glue:** C-interop/JNI/C-FFI layer that spawns the Go server process and initializes it via a MethodChannel.
* **Bridge Layer:** A local-loopback WebSocket server (for bidirectional communication). Runs on `127.0.0.1` at an ephemeral port.
* **Dart Layer:** A mirror of the Pion API using async proxies to the Go server. All objects inherit from `PionResource`.

### 2.2 Security & Networking

* **Interface:** `127.0.0.1` (Localhost) only. No external network access.
* **Authentication:** A 32-character hex `Session-Token` (UUID v4) generated by Go on init and passed to Dart. Included via `X-Pion-Token` header on WebSocket handshake.
* **Port Selection:** Go binds to port `0` (ephemeral) on startup. The assigned port is returned to Dart via MethodChannel.
* **Timeout:** Resources are garbage collected after 300 seconds of inactivity (no messages) unless explicitly closed.

### 2.3 Data Flow


## 3. Go Core (The Server)

### 3.1 Initialization

On startup, the Go binary MUST:

1. **Create a Registry:** Initialize a thread-safe map `Handle -> WebRTC Resource`.
2. **Bind WebSocket Server:** Listen on `127.0.0.1:0` (ephemeral port).
3. **Generate Session Token:** Create a UUID v4, convert to hex string (32 chars, no hyphens). Store in memory.
4. **Return to Caller:** Via stdout, emit JSON: `{"port": <int>, "token": "<32-char-hex>"}`

The platform glue (JNI/FFI/C) parses this output and passes it to Dart.

### 3.2 Resource Registry

**Purpose:** Track all active WebRTC objects (PeerConnections, DataChannels).

**Data Structure:**
```go
type Registry struct {
  mu        sync.RWMutex
  resources map[string]interface{}       // handle -> resource
  lastSeen  map[string]time.Time         // handle -> last activity
}

// Handle format: UUID v4 hex string (e.g., "550e8400e29b41d4a716446655440000")
```

**Operations:**
- **Register:** Generate UUID v4, store object, initialize last-seen timestamp.
- **Lookup:** Thread-safe read with token validation.
- **Delete:** Remove handle, close underlying Pion object (if applicable).
- **Cleanup:** Background goroutine runs every 30 seconds, deletes handles older than 300 seconds (no messages received).

### 3.3 Session Token

**Generation:**
- UUID v4 (random).
- Hex-encoded (32 characters, lowercase, no hyphens).
- Example: `f47ac10b58cc4372a567512a3c4a4e5b`

**Validation:**
- Required header: `X-Pion-Token: <token>` on WebSocket handshake.
- Checked once on upgrade; stored per connection.
- If missing or incorrect, close connection with HTTP 401.

### 3.4 Error Schema & Handling

**All error responses** use a consistent MessagePack format:

```json
{
  "type": "error",
  "id": <request_id>,
  "data": {
    "code": "ERROR_CODE",
    "message": "Human-readable description",
    "fatal": false,
    "handle": "optional-uuid-of-affected-object"
  }
}
```

**Error Codes:**

| Code | Meaning | Recoverable |
|------|---------|-------------|
| `INVALID_TOKEN` | Session token missing/invalid | Yes |
| `INVALID_REQUEST` | Malformed MessagePack or missing fields | Yes |
| `NOT_FOUND` | Handle does not exist | Yes |
| `INVALID_STATE` | Operation not allowed in current state | Yes |
| `INTERNAL_ERROR` | Go error (Pion library error) | Yes |
| `FATAL_PANIC` | Go panic caught by middleware | No |
| `OPERATION_TIMEOUT` | Operation took too long | Yes |
| `CONNECTION_LOST` | WebSocket disconnected unexpectedly | No |

**Panic Recovery Middleware:**
- Wraps all message handlers with `defer recover()`.
- On panic, serialize as `{type: "error", data: {code: "FATAL_PANIC", message: "<panic>", fatal: true}}`.
- Log to stdout for debugging.

### 3.5 WebSocket Message Format

**All communication uses MessagePack serialization.**

**Request/Response Structure:**
```dart
class WsMessage {
  String type;              // Operation or event type
  int id;                   // Request/response ID (for matching)
  String? handle;           // Resource handle (optional)
  Map<String, dynamic> data; // Operation payload or response data
}
```

**MessagePack Encoding:**
The Dart/Go structures are serialized directly as MessagePack binary. No intermediate JSON.

Example (MessagePack hex):
```
84                          // map with 4 entries
a4 74 79 70 65            // "type"
a8 70 63 3a 63 72 65 61 74 65 // "pc:create"
a2 69 64                   // "id"
01                         // 1
a6 68 61 6e 64 6c 65      // "handle"
c0                         // null
a4 64 61 74 61            // "data"
82                         // map with 2 entries
...
```

**Request ID Matching:**
- Dart generates unique request IDs (e.g., auto-incrementing or UUID).
- Go responds with the same `id` so Dart can match requests to responses.
- Allows pipelining multiple requests without blocking.

**Key Classes:**

- **`WebSocketConnection`** — Low-level WebSocket + MessagePack serialization
- **`EventDispatcher`** — Routes incoming events to resources by handle
- **`PionResource`** — Base class for all WebRTC objects (PC, DC)
- **`PionPeerConnection`** — Wrapper around a Pion PeerConnection
- **`PionDataChannel`** — Wrapper around a Pion DataChannel

### 4.2 PionBridge: Main Entry Point

**`PionBridge`** is the single entry point for the Dart layer. It:
1. Manages the WebSocket connection
2. Handles initialization (starting Go server, connecting)
3. Provides factory methods to create PeerConnections
4. Manages cleanup on shutdown

**Initialization:**
```dart
final pion = await PionBridge.initialize();
```

Under the hood:
1. Call MethodChannel `io.filemingo.pionbridge` method `startServer()`.
2. MethodChannel returns `{port: int, token: String}`.
3. Establish WebSocket connection to `ws://127.0.0.1:<port>/` with header `X-Pion-Token: <token>`.
4. Set up MessagePack encoder/decoder and request/response dispatcher.
5. Return initialized `PionBridge` instance.

**Shutdown:**
```dart
await pion.close();
```

This closes the WebSocket and cleans up resources.

### 4.3 PionBridge Implementation

```dart
class PionBridge {
  late WebSocketConnection _connection;
  late EventDispatcher _dispatcher;

  // Private constructor
  PionBridge._();

  /// Initialize PionBridge: start Go server and open WebSocket connection.
  ///
  /// Returns a fully initialized PionBridge instance ready to create PeerConnections.
  static Future<PionBridge> initialize() async {
    final pion = PionBridge._();
    await pion._init();
    return pion;
  }

  /// Internal initialization
  Future<void> _init() async {
    // Start Go server via MethodChannel
    final platform = const MethodChannel('io.filemingo.pionbridge');
    final result = await platform.invokeMethod('startServer');
    final port = result['port'] as int;
    final token = result['token'] as String;

    // Set up WebSocket connection and event dispatcher
    _dispatcher = EventDispatcher();
    _connection = WebSocketConnection(
      onMessage: _dispatcher.broadcast,
      onDisconnect: () {
        // Connection lost — all pending requests are already failed.
        // App code should catch PionException with code CONNECTION_LOST.
      },
    );

    // Connect to the Go server
    await _connection.connect(
      'ws://127.0.0.1:$port/',
      token: token,
    );
  }

  /// Create a new PeerConnection.
  Future<PionPeerConnection> createPeerConnection({
    List<IceServer>? iceServers,
    String bundlePolicy = 'balanced',
    String rtcpMuxPolicy = 'require',
  }) async {
    final response = await _connection.request('pc:create', null, {
      'ice_servers': iceServers?.map((s) => s.toMap()).toList() ?? [],
      'bundle_policy': bundlePolicy,
      'rtcp_mux_policy': rtcpMuxPolicy,
    });

    return PionPeerConnection(
      response['handle'],
      _connection,
      _dispatcher,
    );
  }

  /// Close the bridge and shut down the Go server.
  ///
  /// Closes the WebSocket connection. The Go process will exit gracefully.
  Future<void> close() async {
    await _connection.close();
  }
}
```

### 4.4 WebSocket Connection Manager

**`WebSocketConnection`** manages the low-level WebSocket communication:
- Opens/closes connection
- Serializes outgoing messages to MessagePack
- Deserializes incoming messages from MessagePack
- Matches responses to pending requests by ID
- Routes events to the `EventDispatcher`
- Detects disconnection and fails all pending requests

```dart
class WebSocketConnection {
  late WebSocket ws;
  late MessagePackCodec codec;

  final Map<int, Completer<Map<String, dynamic>>> pendingRequests = {};
  final Function(WsMessage) onMessage; // Callback to EventDispatcher
  final void Function()? onDisconnect; // Called when WebSocket closes unexpectedly

  int nextRequestId = 1;
  bool _connected = false;

  WebSocketConnection({required this.onMessage, this.onDisconnect});

  bool get isConnected => _connected;

  Future<void> connect(String url, String token) async {
    ws = await WebSocket.connect(
      url,
      headers: {'X-Pion-Token': token},
    );
    _connected = true;

    // Listen for incoming messages
    ws.listen(
      (message) => _handleMessage(message),
      onError: (error) => _handleDisconnect('WebSocket error: $error'),
      onDone: () => _handleDisconnect('WebSocket closed'),
    );
  }

  void _handleDisconnect(String reason) {
    _connected = false;

    // Fail all pending requests
    final pending = Map.of(pendingRequests);
    pendingRequests.clear();
    for (final completer in pending.values) {
      completer.completeError(
        PionException('CONNECTION_LOST', reason, fatal: true),
      );
    }

    onDisconnect?.call();
  }

  void _handleMessage(dynamic message) {
    // message is Uint8List (binary MessagePack)
    final decoded = codec.decode(message);
    final wsMsg = WsMessage.fromMap(decoded);

    // Is this a response to a pending request?
    if (wsMsg.type.endsWith(':ack') || wsMsg.type == 'error') {
      final completer = pendingRequests.remove(wsMsg.id);
      if (completer != null) {
        if (wsMsg.type == 'error') {
          completer.completeError(PionException.fromWsMessage(wsMsg));
        } else {
          completer.complete(wsMsg.data);
        }
      }
    } else {
      // Event - pass to listener (EventDispatcher)
      onMessage(wsMsg);
    }
  }

  /// Send a request and wait for response.
  Future<Map<String, dynamic>> request(
    String type,
    String? handle,
    Map<String, dynamic> data,
  ) async {
    if (!_connected) {
      throw PionException('CONNECTION_LOST', 'WebSocket is not connected', fatal: true);
    }

    final id = nextRequestId++;
    final msg = WsMessage(
      type: type,
      id: id,
      handle: handle,
      data: data,
    );

    final completer = Completer<Map<String, dynamic>>();
    pendingRequests[id] = completer;

    // Send as MessagePack binary
    ws.add(codec.encode(msg.toMap()));

    // Timeout after 30 seconds
    return completer.future.timeout(
      const Duration(seconds: 30),
      onTimeout: () => throw PionException('OPERATION_TIMEOUT', 'Request timed out'),
    );
  }

  Future<void> close() async {
    _connected = false;
    await ws.close();
  }
}

class WsMessage {
  final String type;
  final int id;
  final String? handle;
  final Map<String, dynamic> data;

  WsMessage({
    required this.type,
    required this.id,
    this.handle,
    required this.data,
  });

  Map<String, dynamic> toMap() => {
    'type': type,
    'id': id,
    if (handle != null) 'handle': handle,
    'data': data,
  };

  factory WsMessage.fromMap(Map<String, dynamic> map) => WsMessage(
    type: map['type'],
    id: map['id'],
    handle: map['handle'],
    data: map['data'] ?? {},
  );
}
```

### 4.5 EventDispatcher

**`EventDispatcher`** routes incoming events to listening objects by `handle`. Each resource (PeerConnection, DataChannel) registers a listener with its handle. The full `WsMessage` is passed to listeners so they can filter on both `type` and `data`.

```dart
class EventDispatcher {
  final Map<String, StreamController<WsMessage>> listeners = {};

  /// Broadcast an event to all listeners for this handle.
  void broadcast(WsMessage message) {
    final handle = message.handle;
    if (handle != null && listeners.containsKey(handle)) {
      listeners[handle]!.add(message);
    }
  }

  /// Get a stream of events for a specific handle.
  Stream<WsMessage> listen(String handle) {
    if (!listeners.containsKey(handle)) {
      listeners[handle] = StreamController<WsMessage>.broadcast();
    }
    return listeners[handle]!.stream;
  }

  /// Unsubscribe and clean up a handle's listener.
  void unsubscribe(String handle) {
    listeners[handle]?.close();
    listeners.remove(handle);
  }
}
```

### 4.6 PionResource Base Class

All Dart WebRTC objects inherit from `PionResource`, which provides:
- A reference to the shared `WebSocketConnection`
- A reference to the shared `EventDispatcher`
- A handle (UUID) identifying the resource on the Go side
- A `request()` method to send operations to Go
- An `onEvent()` method to listen to events for this resource
- A `close()` method to clean up

All Dart WebRTC objects inherit from `PionResource`:

```dart
abstract class PionResource {
  final String handle;
  final WebSocketConnection connection;
  final EventDispatcher dispatcher;

  PionResource(this.handle, this.connection, this.dispatcher);

  /// Send an operation request and wait for response.
  Future<Map<String, dynamic>> request(
    String type,
    Map<String, dynamic> data,
  ) {
    return connection.request(type, handle, data);
  }

  /// Subscribe to events for this resource.
  Stream<WsMessage> onEvent() {
    return dispatcher.listen(handle);
  }

  /// Close and clean up this resource.
  Future<void> close() async {
    await request('resource:delete', {});
    dispatcher.unsubscribe(handle);
  }
}
```

```dart
class PionException implements Exception {
  final String code;
  final String message;
  final bool fatal;
  final String? handle;

  PionException(
    this.code,
    this.message, {
    this.fatal = false,
    this.handle,
  });

  factory PionException.fromWsMessage(WsMessage msg) {
    final error = msg.data;
    return PionException(
      error['code'],
      error['message'],
      fatal: error['fatal'] ?? false,
      handle: error['handle'],
    );
  }

  @override
  String toString() => 'PionException($code): $message';
}
```

### 4.8 PionPeerConnection

```dart
class PionPeerConnection extends PionResource {
  late Stream<IceCandidate> _onIceCandidate;
  late Stream<void> _onIceGatheringComplete;
  late Stream<PionDataChannel> _onDataChannel;
  late Stream<ConnectionState> _onConnectionStateChange;

  PionPeerConnection(
    String handle,
    WebSocketConnection connection,
    EventDispatcher dispatcher,
  ) : super(handle, connection, dispatcher) {
    _setupStreams();
  }

  void _setupStreams() {
    _onIceCandidate = onEvent()
        .where((msg) => msg.type == 'event:iceCandidate')
        .map((msg) {
          return IceCandidate(
            candidate: msg.data['candidate'],
            sdpMid: msg.data['sdp_mid'],
            sdpMlineIndex: msg.data['sdp_mline_index'],
          );
        });

    _onIceGatheringComplete = onEvent()
        .where((msg) => msg.type == 'event:iceGatheringComplete')
        .map((_) => null);

    _onDataChannel = onEvent()
        .where((msg) => msg.type == 'event:dataChannel')
        .map((msg) {
          return PionDataChannel(
            msg.data['dc_handle'],
            connection,
            dispatcher,
            label: msg.data['label'] ?? '',
          );
        });

    _onConnectionStateChange = onEvent()
        .where((msg) => msg.type == 'event:connectionStateChange')
        .map((msg) {
          return ConnectionState.fromString(msg.data['state'] as String);
        });
  }

  Stream<IceCandidate> get onIceCandidate => _onIceCandidate;
  Stream<void> get onIceGatheringComplete => _onIceGatheringComplete;
  Stream<PionDataChannel> get onDataChannel => _onDataChannel;
  Stream<ConnectionState> get onConnectionStateChange => _onConnectionStateChange;

  Future<String> createOffer({Map<String, dynamic>? options}) async {
    final response = await request('pc:offer', {
      'offer_options': options ?? {},
    });
    return response['sdp'];
  }

  Future<String> createAnswer({Map<String, dynamic>? options}) async {
    final response = await request('pc:answer', {
      'answer_options': options ?? {},
    });
    return response['sdp'];
  }

  Future<void> setLocalDescription(String sdp, String type) async {
    await request('pc:setLocalDesc', {
      'sdp': sdp,
      'type': type,
    });
  }

  Future<void> setRemoteDescription(String sdp, String type) async {
    await request('pc:setRemoteDesc', {
      'sdp': sdp,
      'type': type,
    });
  }

  Future<void> addIceCandidate(IceCandidate candidate) async {
    await request('pc:addIce', {
      'candidate': candidate.candidate,
      'sdp_mid': candidate.sdpMid,
      'sdp_mline_index': candidate.sdpMlineIndex,
    });
  }

  Future<PionDataChannel> createDataChannel(
    String label, {
    bool ordered = true,
    int? maxRetransmits,
    int? maxPacketLifetimeMs,
  }) async {
    final response = await request('pc:createDc', {
      'label': label,
      'options': {
        'ordered': ordered,
        if (maxRetransmits != null) 'max_retransmits': maxRetransmits,
        if (maxPacketLifetimeMs != null) 'max_packet_lifetime_ms': maxPacketLifetimeMs,
      },
    });
    return PionDataChannel(response['dc_handle'], connection, dispatcher);
  }
}
```

### 4.9 PionDataChannel

```dart
class PionDataChannel extends PionResource {
  final String label;
  late Stream<DataChannelMessage> _onMessage;
  late Stream<void> _onOpen;
  late Stream<void> _onClose;

  PionDataChannel(
    String handle,
    WebSocketConnection connection,
    EventDispatcher dispatcher, {
    this.label = '',
  }) : super(handle, connection, dispatcher) {
    _setupStreams();
  }

  void _setupStreams() {
    _onMessage = onEvent()
        .where((msg) => msg.type == 'event:dataChannelMessage')
        .map((msg) => DataChannelMessage(
          data: msg.data['data'],
          isBinary: msg.data['is_binary'] ?? false,
        ));

    _onOpen = onEvent()
        .where((msg) => msg.type == 'event:dataChannelOpen')
        .map((_) => null);

    _onClose = onEvent()
        .where((msg) => msg.type == 'event:dataChannelClose')
        .map((_) => null);
  }

  Stream<DataChannelMessage> get onMessage => _onMessage;
  Stream<void> get onOpen => _onOpen;
  Stream<void> get onClose => _onClose;

  /// Send text data through the data channel.
  Future<void> send(String data) async {
    await request('dc:send', {
      'data': data,
      'is_binary': false,
    });
  }

  /// Send raw binary data through the data channel.
  Future<void> sendBinary(List<int> data) async {
    await request('dc:send', {
      'data': base64Encode(data),
      'is_binary': true,
    });
  }
}
```

## 5. WebSocket Operations Reference

All operations are MessagePack-encoded. Format: `{type, id, handle?, data}`

### Operations (Dart → Go)

#### `init` — Verify session (optional)

Sent after WebSocket connection to verify the session is active. Authentication is already
handled via the `X-Pion-Token` header on handshake, so this is optional — useful for
confirming the server is ready and the token was accepted.

**Request:**
```json
{"type": "init", "id": 1, "data": {}}
```

**Response:**
```json
{"type": "init:ack", "id": 1, "data": {"version": "1.0.0"}}
```

---

#### `pc:create` — Create PeerConnection

**Request:**
```json
{
  "type": "pc:create",
  "id": 2,
  "data": {
    "ice_servers": [{"urls": ["stun:stun.l.google.com:19302"]}],
    "bundle_policy": "balanced",
    "rtcp_mux_policy": "require"
  }
}
```

**Response:**
```json
{"type": "pc:create:ack", "id": 2, "data": {"handle": "...", "state": "new"}}
```

#### `resource:delete` — Explicit Resource Cleanup

Closes the underlying Pion object (if not already closed) **and** removes it from the registry.
This is the recommended way to clean up resources. `pc:close` and `dc:close` only close the
underlying Pion object but leave the handle in the registry (useful if you still need to read
final state or drain events). `resource:delete` combines both: close + deregister.

**Request:**
```json
{"type": "resource:delete", "id": 11, "handle": "pc-or-dc-handle", "data": {}}
```

**Response:**
```json
{"type": "resource:delete:ack", "id": 11, "handle": "...", "data": {}}
```

## 6. Resource Lifecycle & Cleanup

**Implicit Cleanup (Timeout):**
- Every 30 seconds, the Go server scans the registry.
- Any resource with `time.Now() - lastSeen > 300 seconds` is automatically closed and deleted.
- `lastSeen` is updated on every message (operation or event) involving the handle.

**Explicit Cleanup:**
- Dart can call `resource:delete` to immediately close and remove.
- Recommended on app exit or when the user stops using a connection.

**Reference Counting:**
- When a PeerConnection is deleted, all its DataChannels are also deleted.

## 8. Example Usage (Dart)

**Offerer side:**
```dart
void offerer() async {
  final pion = await PionBridge.initialize();

  final pc = await pion.createPeerConnection(
    iceServers: [IceServer(urls: ['stun:stun.l.google.com:19302'])],
  );

  // Listen to ICE candidates and forward to remote
  pc.onIceCandidate.listen((candidate) {
    sendToRemote({'type': 'ice', 'candidate': candidate});
  });

  pc.onConnectionStateChange.listen((state) {
    print('Connection state: $state');
  });

  // Create offer
  final offer = await pc.createOffer();
  await pc.setLocalDescription(offer, 'offer');
  sendToRemote({'type': 'offer', 'sdp': offer});

  // Receive answer from remote
  final remoteAnswer = await receiveFromSignaling();
  await pc.setRemoteDescription(remoteAnswer['sdp'], 'answer');

  // Add remote ICE candidates
  receiveRemoteCandidates().listen((candidate) async {
    await pc.addIceCandidate(candidate);
  });

  // Create data channel
  final dc = await pc.createDataChannel('chat');

  dc.onOpen.listen((_) => print('Data channel open'));
  dc.onMessage.listen((msg) {
    if (msg.isBinary) {
      print('Binary: ${msg.binaryData.length} bytes');
    } else {
      print('Text: ${msg.data}');
    }
  });

  await dc.send('Hello, peer!');
}
```

**Answerer side:**
```dart
void answerer() async {
  final pion = await PionBridge.initialize();

  final pc = await pion.createPeerConnection(
    iceServers: [IceServer(urls: ['stun:stun.l.google.com:19302'])],
  );

  pc.onIceCandidate.listen((candidate) {
    sendToRemote({'type': 'ice', 'candidate': candidate});
  });

  // Receive offer from remote
  final remoteOffer = await receiveFromSignaling();
  await pc.setRemoteDescription(remoteOffer['sdp'], 'offer');

  // Create answer
  final answer = await pc.createAnswer();
  await pc.setLocalDescription(answer, 'answer');
  sendToRemote({'type': 'answer', 'sdp': answer});

  // Listen for incoming data channels from the offerer
  pc.onDataChannel.listen((dc) {
    dc.onOpen.listen((_) => print('Data channel open'));
    dc.onMessage.listen((msg) => print('Received: ${msg.data}'));
  });

  // Clean up when done
  await pion.close();
}
```

## 9. MessagePack Serialization

**Dependencies:**
- **Go:** `github.com/vmihailenco/msgpack/v5`
- **Dart:** `msgpack_dart` or `messagepack` package

**Example (Dart):**
```dart
import 'package:msgpack_dart/msgpack_dart.dart';

final codec = Codec();

// Encode
final wsMsg = {'type': 'pc:create', 'id': 1, 'handle': null, 'data': {}};
final bytes = codec.encode(wsMsg);
ws.add(bytes); // Send binary

// Decode
final incoming = ws.listen((message) {
  final decoded = codec.decode(message);
  print(decoded); // Map<String, dynamic>
});
```

## Appendix: Session Token Format

Session tokens are 32-character lowercase hex strings (UUID v4):
```
f47ac10b58cc4372a567512a3c4a4e5b
```

Generated once per initialization. Included in WebSocket handshake:
```
GET / HTTP/1.1
X-Pion-Token: f47ac10b58cc4372a567512a3c4a4e5b
Upgrade: websocket
Connection: Upgrade
```
